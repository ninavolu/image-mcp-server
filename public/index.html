MCP Connector

Your image library,
inside Claude.

A read-only MCP server that lets Claude search and use your curated image library — stored in Supabase, tagged by AI, accessible anywhere.

Documentation

What it does

Image Library MCP connects Claude to a curated database of images stored in Supabase. Claude can search by purpose (hero, landing-page, product, etc.) and tags to find the right image for any task.

Tools

search_images
read-only
Search images by purpose and/or tags. Returns name, URL, purpose, tags, and description for each match.
list_images
read-only
List all images in the library with pagination support.

Example prompts

Find me a dark minimal hero image for a landing page
Search my image library for nature photos tagged "outdoor"
Build a landing page using the best hero image from my library
List all product images I have available

Setup

Add the following to your claude_desktop_config.json:

{ "mcpServers": { "image-library": { "url": "https://image-mcp-server-production.up.railway.app/sse" } } }

Authentication

This server uses OAuth 2.1. When you first connect, you'll be prompted to sign in with your email or Google account via Clerk.

Privacy Policy

Last updated: May 2026

Data we collect

We collect your email address and name when you authenticate via Clerk (our OAuth provider). We do not collect any other personal data.

How we use your data

Your account information is used solely to authenticate you to the MCP server. We do not use it for marketing, analytics, or any other purpose.

Data storage

Authentication is handled by Clerk. Image metadata is stored in Supabase (PostgreSQL). Image files are stored in Supabase Storage. No personal data is stored in our image database.

Third-party services

We use Clerk for authentication (clerk.com/privacy) and Supabase for database and storage (supabase.com/privacy).

Data retention

Your account data is retained as long as your account is active. You may request deletion at any time by contacting us.
